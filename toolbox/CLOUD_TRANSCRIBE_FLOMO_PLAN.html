Cardputer 录音自动上传、转写与 flomo 同步方案

目标：Cardputer 录完音后自动上传到阿里云服务器，服务器转成文字，再自动写入 flomo。本文档用于后续开发对齐。

目标 总链路 准备清单 服务器 设备端 实施阶段 安全 待确认

一、目标与边界

项目决定
设备端职责只负责录音、本地保存、排队上传 WAV。不要在 Cardputer 上转文字，也不要把 flomo webhook 放进固件。
服务器职责接收录音、保存文件、调用阿里云语音识别、生成文本、发送到 flomo。
flomo 职责只接收服务器发来的文字 memo。flomo 自带语音输入不作为后台音频转写 API 使用。
性能原则上传必须后台化、可重试，不能影响录音、播放和列表操作。

二、总链路

Cardputer 录音保存 WAV
  -> Wi-Fi 上传到阿里云服务器 /upload
  -> 服务器保存 uploads/REC_XXXX.wav
  -> 调用阿里云 Paraformer / DashScope 转文字
  -> 生成 notes/REC_XXXX.txt 和 JSON 状态
  -> 调用 flomo Incoming Webhook 写入 flomo
  -> 服务器返回上传/转写/发送结果
  -> Cardputer 标记本地录音为已上传
推荐先跑通“上传并保存 WAV”，再接转写，最后接 flomo。不要一开始就做全链路，否则排错会很乱。

三、准备清单

状态事项说明
[ ]阿里云服务器需要公网 IP 或域名，能 SSH 登录，建议 Ubuntu / Debian。
[ ]Node.js 运行环境建议 Node.js 20 LTS，用 Express 写轻量服务。
[ ]阿里云语音识别 API Key开通 DashScope / Paraformer，拿到 DASHSCOPE_API_KEY。
[ ]flomo PRO开通后获取 Incoming Webhook，形如 https://flomoapp.com/iwh/...。
[ ]Cardputer Wi-Fi 信息SSID 和密码只放本地配置或后续配网方案，不提交 GitHub。
[ ]服务器上传密钥自定义一个 UPLOAD_TOKEN，Cardputer 上传时带上，避免接口裸奔。

四、服务器设计

目录结构建议

/opt/cardputer-voice/
├── server.js
├── package.json
├── uploads/       # 原始 WAV
├── transcripts/   # 转写文本
├── jobs/          # 状态 JSON
└── logs/

环境变量

PORT=3000
UPLOAD_TOKEN=一串随机长密码
DASHSCOPE_API_KEY=阿里云语音识别key
FLOMO_WEBHOOK_URL=https://flomoapp.com/iwh/...
PUBLIC_BASE_URL=https://你的域名

接口规划

接口方法作用
/healthGET服务器健康检查，Cardputer 可用它测试网络。
/uploadPOST上传 WAV，Header 带 X-Upload-Token。
/jobs/:idGET查看某条录音的处理状态，调试用。

上传请求格式

POST /upload
Headers:
  X-Upload-Token: UPLOAD_TOKEN
  X-Device-Id: cardputer-001
  X-Recording-Name: REC_0123.wav
Content-Type: audio/wav

Body:
  WAV bytes

flomo 笔记格式建议

#Cardputer语音

转写内容……

---
录音：REC_0123.wav
时间：2026-05-23 21:30
来源：Cardputer
状态：自动转写

五、Cardputer 端设计

模块做法
上传时机录音保存完成后加入上传队列；优先不阻塞 UI。
失败处理上传失败保留本地 WAV，下次开机/进入列表/空闲时重试。
状态记录后续可在 SD 写 /UPLOAD/status.txt，记录已上传编号。
目录关系普通、快捷、重要录音都可以上传；上传不应改变三列目录分类。
性能约束播放中不上传；录音中不上传；只在空闲、列表或保存后短窗口尝试。

六、实施阶段

状态阶段验收标准
[ ]阶段 1：服务器最小上传接口用电脑 curl 上传一个 WAV，服务器保存到 uploads/。
[ ]阶段 2：Cardputer 上传 WAV录音保存后能上传到服务器；失败不会影响本地录音。
[ ]阶段 3：接阿里云 Paraformer服务器能把上传的 WAV 转成文字，并保存 .txt。
[ ]阶段 4：接 flomo Webhook转写文字自动出现在 flomo。
[ ]阶段 5：上传队列与状态断网后能重试；已上传录音不会重复发 flomo。
[ ]阶段 6：清理策略可安全清空普通 /REC，不影响 /SHORTCUT 与 /IMPORTANT。

七、安全原则

不要把 DASHSCOPE_API_KEY、FLOMO_WEBHOOK_URL、UPLOAD_TOKEN、Wi-Fi 密码提交到 GitHub。
  • flomo webhook 只放服务器环境变量。
  • 阿里云 API Key 只放服务器环境变量。
  • Cardputer 固件最多只保存服务器上传地址和上传 token；如果后续能做本地配置文件，就不要写死进源码。
  • 服务器接口必须校验 X-Upload-Token。
  • 服务器日志不要打印完整 webhook 和 API Key。
