编译 & 烧录流程

一键脚本

powershell -File C:\cardputer\tools\build_and_flash.ps1
2026-05-23 实测：当前本地 build_and_flash.ps1 曾因编码损坏报 PowerShell 字符串结束符错误。若脚本失败，按下面手动三步走；必要时用 COMPILE_FLASH.md 的 UTF-8 版本重写脚本。

手动三步

# 1. 同步源码
Copy-Item -Force "D:\github仓库同步\小机器\toolbox\toolbox.ino" `
          "C:\cardputer\sketches\toolbox\toolbox.ino"

# 2. 编译
$cli = "C:\cardputer\tools\arduino-cli\arduino-cli.exe"
$proc = Start-Process -FilePath $cli `
    -ArgumentList "compile --fqbn esp32:esp32:m5stack_cardputer --build-path C:\cardputer\build_fresh\out C:\cardputer\sketches\toolbox" `
    -RedirectStandardOutput C:\cardputer\tools\tb.out.log `
    -RedirectStandardError  C:\cardputer\tools\tb.err.log `
    -NoNewWindow -PassThru -Wait
Write-Host "Exit: $($proc.ExitCode)"   # 0=成功

# 3. 烧录（先确认设备唤醒，按一下键盘）
$proc = Start-Process -FilePath $cli `
    -ArgumentList "upload --fqbn esp32:esp32:m5stack_cardputer --port COM3 --input-dir C:\cardputer\build_fresh\out" `
    -RedirectStandardOutput C:\cardputer\tools\tb.out.log `
    -RedirectStandardError  C:\cardputer\tools\tb.err.log `
    -NoNewWindow -PassThru -Wait

代码架构（toolbox.ino）

主要函数

函数说明
setup()初始化M5、SD、音频，加载hotkeys，进入loop
loop()开机/唤醒后若 autoRecordPending 为真，根据 wakeAction 进入录音或列表；否则进入息屏等待 Space/Enter
recordingScreen()真正录音流程。调用后创建WAV并开始写SD
playbackScreen(int recNum)回放界面。播放自然结束后自动返回列表；Enter 暂停/继续；退格返回列表；;/.切换上一条/下一条；长按 Del 可删除当前录音
listScreen()录音列表，;/.翻页，回车播放，Ctrl+键绑定快捷键，Alt降噪
drawRecCanvas(...)录音Canvas帧：AB双波形 + 计时器 + 电量
drawPlaybackCanvas(...)播放Canvas帧：A线为播放时间轴，B线为当前播放缓冲跳动
afterRecordingFlow(int recNum)录音结束后默认回列表；快捷键仍可直接播放
listFlow(int n)列表界面的完整流程控制（含从列表→录音→再返回的循环）
playFlow(int n)播放流程控制
previewPlaybackWaveStep(...) / updatePlaybackWaveBars(...)播放时后台逐格预览 A线，并用当前音频缓冲增量修正；不再进播放页前完整扫描 WAV

返回码常量

常量值含义
R_BACK0退格：返回上一层
R_RECORD1空格：去录音
R_PLAY2播放键：切换播放（g_nextPlay存目标编号）
R_LIST3列表键：进入列表
R_NOISE4Alt键：降噪
R_DELETE5播放页 Del 长按确认后删除当前录音

Canvas / Display

  • 录音动态画面用 M5Canvas（240×135），pushSprite(0,0) 推到屏幕
  • 右侧三段标签栏已取消，减少界面切换重绘与视觉噪音
  • 独立启动页已清理，开机/空格唤醒后的第一次 loop() 直接开始真正录音；息屏回车唤醒会直接进入录音列表
  • READY 只作为录音准备帧；真正启动 Mic.record() 后才亮 REC 红点
  • 录音准备会复用已热机的麦克风输入；启动录音缓冲后，松开前只忽略停止事件
  • 开机/唤醒预热也必须走 prepareMicInput() 完整输入链路，避免第一次录音音量偏低
  • 开机/唤醒后的第一次正式录音会 forceMicRearm：强制重建输入链路并丢弃 6 帧稳定缓冲
  • 录音准备阶段会消费 forceMicRearm，避免第一条录音音量偏低
  • 播放自然结束后自动返回列表，不再停在播放页等待按键
  • 录音中 Space 是暂停/继续：暂停时停止写入 SD、计时冻结；继续时丢弃恢复瞬间 2 帧缓冲
  • 息屏键统一为 Esc：列表页 Esc 退出息屏，播放页 Esc 停止播放并息屏，录音中 Esc 保存当前录音后息屏。Cardputer 库里物理 Esc 映射为左上角反引号键
  • 删除操作统一为长按 Del 1.2 秒：录音中删除本条、播放页删除当前录音、列表页删除选中录音；按住显示红色进度条和“删除中”，松开取消
  • 新录音编号用 nextRecHint 缓存和目录扫描获取，不再每次从 REC_0001.wav 顺序探测
  • 播放页也使用 Canvas 双缓冲推屏；A线在扬声器队列空隙里后台多点预览，并随播放缓冲增量修正，统一画成亮绿小波形柱，播放头表示进度，避免进播放页前完整扫描 WAV 造成等待
  • 播放入口不再阻塞等待回车松开；启动按键会被非阻塞忽略，音频队列可以立即开始填充
  • 所有主要按键先给即时反馈再执行慢操作：列表播放显示 PLAY，播放切换显示 PREV/NEXT，暂停/继续显示 PAUSE/PLAY，返回/录音/息屏也会先闪提示
  • 播放流程复用当前录音列表缓存，只有缓存缺失时才重新扫描 SD，减少列表播放和播放中上下切换的等待
  • 录音开始查找新编号时顺手填充录音列表缓存，保存后直接插入新编号，回列表不再重复扫描 SD；保存时先显示 SAVE
  • 列表删除走满进度后显示 DEL，删除后只从内存列表移除该条；只有删除了绑定快捷键时才重写 keys.txt
  • 统一命名：上半区叫 A线 / 轨道线 / Timeline A；下半区叫 B线 / 监听线 / Monitor B
  • 交互统一为：息屏 Space 录音、息屏 Enter 进列表；录音中 Space 暂停/继续、回车保存进列表、Esc 保存并息屏；列表回车播放选中录音；播放中回车暂停/继续、退格回列表、;/.切换上一条/下一条、Esc 息屏
  • 反斜杠键不再作为主流程按键，避免与回车/退格形成重复入口

⛔ 约束与禁令

禁止原因
升级 esp32 core 到 3.2.x / 3.3.xMic.begin() 在新版崩溃，只能用 3.1.3-cn
在中文路径下编译arduino-cli 子进程会报路径错误
直接编辑 C:\cardputer\sketches\...ino同步时会被覆盖
添加 efontCN_24（24px中文字体）占~350KB，Flash直接超限；用 efontCN_12 或 Font4
C++ 默认参数写两遍声明与定义只能有一处写 = default，否则编译报 -fpermissive
用 compile --upload 一步完成难以排查错误，分步更安全
把 arduino-cli 加入PATH后裸调用可能与系统其他版本冲突，始终用完整路径

恢复esp32 3.1.3核心

$cli = "C:\cardputer\tools\arduino-cli\arduino-cli.exe"
& $cli config dump
& $cli config add board_manager.additional_urls `
    "https://jihulab.com/esp-mirror/espressif/arduino-esp32/-/raw/gh-pages/package_esp32_index_cn.json"
& $cli core update-index
& $cli core install esp32:esp32@3.1.3
若 packages\esp32\tools 还在但 hardware\esp32\3.1.3 缺失，直接重装 esp32:esp32@3.1.3 即可，不必清理 tools。不要使用 /arduino/arduino-esp32/...package_esp32_cn_index.json，2026-05-23 实测返回 401 Unauthorized。
