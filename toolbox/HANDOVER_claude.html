编译 & 烧录流程

一键脚本（推荐）

powershell -File C:\cardputer\tools\build_and_flash.ps1
脚本依次：①同步源码 ②编译（60–120秒）③烧录到COM3。编译失败自动停止，不会烧旧固件。

手动三步

# 1. 同步源码
Copy-Item -Force "D:\github仓库同步\小机器\toolbox\toolbox.ino" `
          "C:\cardputer\sketches\toolbox\toolbox.ino"

# 2. 编译
$cli = "C:\cardputer\tools\arduino-cli\arduino-cli.exe"
$proc = Start-Process -FilePath $cli `
    -ArgumentList "compile --fqbn esp32:esp32:m5stack_cardputer --build-path C:\cardputer\build_fresh\out C:\cardputer\sketches\toolbox" `
    -RedirectStandardOutput C:\cardputer\tools\tb.out.log `
    -RedirectStandardError  C:\cardputer\tools\tb.err.log `
    -NoNewWindow -PassThru -Wait
Write-Host "Exit: $($proc.ExitCode)"   # 0=成功

# 3. 烧录（先确认设备唤醒，按一下键盘）
$proc = Start-Process -FilePath $cli `
    -ArgumentList "upload --fqbn esp32:esp32:m5stack_cardputer --port COM3 --input-dir C:\cardputer\build_fresh\out" `
    -RedirectStandardOutput C:\cardputer\tools\tb.out.log `
    -RedirectStandardError  C:\cardputer\tools\tb.err.log `
    -NoNewWindow -PassThru -Wait

代码架构（toolbox.ino）

主要函数

函数说明
setup()初始化M5、SD、音频，加载hotkeys，进入loop
loop()主屏（drawHome），等待按键：Space→录音，Enter→播放，\→列表，快捷键→直接播放
recordingScreen(bool showIdle)录音界面。showIdle=true先显示待机态等第二次Space；false直接录音（主屏调用）
playbackScreen(int recNum)回放界面。播放自然结束后停留在界面，等用户按键离开
listScreen()录音列表，;/.翻页，回车播放，Ctrl+键绑定快捷键，N降噪
drawHome()主屏（录音待机态视觉），TAB_REC高亮
drawTabBar(int active)直接在Display上画右侧标签栏（不用Canvas），active=0/1/2
drawRecCanvas(...)录音Canvas帧：AB双波形 + 计时器 + 电量
drawWaveProgress(...)回放进度波形（连线样式，绿色=已播/暗=未播，白色播放头）
afterRecordingFlow(int recNum)录音结束后路由：g_afterRecord决定去列表/播放
listFlow(int n)列表界面的完整流程控制（含从列表→录音→再返回的循环）
playFlow(int n)播放流程控制
loadWaveBars(File &f)从WAV文件采样填充waveBars[]（int8_t有符号，保留极性）

返回码常量

常量值含义
R_BACK0退格：返回上一层
R_RECORD1空格：去录音
R_PLAY2播放键：切换播放（g_nextPlay存目标编号）
R_LIST3列表键：进入列表
R_NOISE4N键：降噪

Canvas vs Display

  • 内容区用 M5Canvas（212×135），pushSprite(0,0) 推到屏幕
  • 标签栏（x 212–239）直接用 M5Cardputer.Display 绘制，Canvas推送不会覆盖它
  • drawTabBar() 在每个界面初始化时调用一次即可

⛔ 约束与禁令

禁止原因
升级 esp32 core 到 3.2.x / 3.3.xMic.begin() 在新版崩溃，只能用 3.1.3-cn
在中文路径下编译arduino-cli 子进程会报路径错误
直接编辑 C:\cardputer\sketches\...ino同步时会被覆盖
添加 efontCN_24（24px中文字体）占~350KB，Flash直接超限；用 efontCN_12 或 Font4
C++ 默认参数写两遍声明与定义只能有一处写 = default，否则编译报 -fpermissive
用 compile --upload 一步完成难以排查错误，分步更安全
把 arduino-cli 加入PATH后裸调用可能与系统其他版本冲突，始终用完整路径

常见编译错误

错误信息原因解决
Platform 'esp32:esp32' not found3.1.3核心丢失重装：arduino-cli core install esp32:esp32@3.1.3（需jihulab镜像）
esp8266-compat.h not found混入了3.3.x核心手动删除 ...\Arduino15\packages\esp32\hardware\esp32\3.3.x
Arduino.h not found构建缓存污染删除 C:\cardputer\build_fresh\out 重编
default argument given for parameter函数声明和定义都写了默认值默认值只保留在前向声明中，定义里去掉
Flash超过100%引入大字体库（efontCN_24等）改用 Font4 或 efontCN_12

恢复esp32 3.1.3核心

$cli = "C:\cardputer\tools\arduino-cli\arduino-cli.exe"
& $cli config add board_manager.additional_urls `
    "https://jihulab.com/esp-mirror/arduino/arduino-esp32/-/raw/gh-pages/package_esp32_cn_index.json"
& $cli core update-index
& $cli core install esp32:esp32@3.1.3
