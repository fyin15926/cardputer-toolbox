# Cardputer 录音工具 — 设计规格文档
# UI & Interaction Design Specification

> 本文档基于用户交互流程图与界面原型草图撰写，作为后续编码实现的设计基准。  
> 屏幕分辨率：240×135 px，ST7789V2，横屏。

> 2026-05-23 简化决策：右侧标签栏方案已取消，固件改为 240px 全宽内容区。开机/空格唤醒后直接进入真正录音，息屏回车唤醒后进入列表；录音中 Space 为暂停/继续，Enter 保存进列表，Esc 保存并息屏；列表中 Enter 播放选中录音，播放中 Enter 暂停/继续、退格回列表、;/.切换上一条/下一条。`READY` 只作为录音准备帧，真正 `Mic.record()` 启动后才显示 `REC` 红点。

> 双线命名：上半区为 **A线 / 轨道线 / Timeline A**；下半区为 **B线 / 监听线 / Monitor B**。录音时 A线是滚动历史、B线是实时输入；播放时 A线是时间轴进度、B线是当前播放缓冲。
> A线和B线之间不绘制额外分隔线，只保留各自的暗基线。

## 1. 整体视觉风格

| 属性 | 值 |
|------|----|
| 背景色 | `#000000`（纯黑） |
| 主色 | `#00FF41`（磷光绿，仿老式 CRT 终端） |

## 2. 全局布局

```
┌────────────────────────────────────────┐
│ READY / REC / REC_0001            65% │
├────────────────────────────────────────┤
│ A线 / 轨道线：录音历史或播放时间轴       │
│ B线 / 监听线：输入监听或播放缓冲         │
├────────────────────────────────────────┤
│ 00:00                                  │
└────────────────────────────────────────┘
```

### 导航原则

- 不再绘制右侧标签栏，所有状态使用 240px 全宽画面。
- 不再有独立主屏；开机和息屏空格唤醒直接录音，息屏回车唤醒进入列表，Esc 是统一息屏键。
- Enter 的语义统一：录音中保存进列表；列表播放选中；播放中暂停/继续。Space 在录音中为暂停/继续，播放中退格返回列表。

### 电量（右上角，叠加在主内容区右上）

- 位置：右上角，坐标约 `(206, 1)`
- 格式：`XX%`，字号 8px，亮绿色
- 电量 ≤ 20% 时变红色并闪烁

## 3. 录音界面

### 3.1 界面布局

```
┌──────────────────────────────────────┐
│ ● REC              65%               │  ← 顶栏 14px 高
├──────────────────────────────────────┤
│                                      │
│   A线：滚动历史                         │
│   B线：实时输入                         │  ← 波形区 93px 高
│                                      │
├──────────────────────────────────────┤
│ 03:34                                │  ← 底栏 26px 高，左下时间
└──────────────────────────────────────┘
```

### 3.2 各元素规格

#### 顶栏
- **红点**：半径 5px 圆，`#FF3B30`，位于 `(8, 7)`，**1Hz 闪烁**（录制中）/ 录制停止后熄灭
- **"REC" 文字**：紧跟红点右侧，亮绿色，等宽字体 font2（10px），录制中显示
- **电量**：右上角，见全局布局

#### 波形区（实时录音波形）
- 区域：`y=16..109`，高度 93px，黑色背景
- A线：**从右向左滚动**（新数据从右侧进入，旧数据向左推出）
- B线：当前麦克风缓冲实时跳动
- 颜色：`#00FF41`
- A线中轴 `A_CY=39`，B线中轴 `B_CY=85`
- 不绘制 A/B 之间的额外分隔线，只保留各自暗基线

#### 时间计数器
- 位置：底部左侧 `(4, 110)`
- 格式：`MM:SS`
- 字号：**最大可用**（约 font4 / font6，高度 18–22px）
- 颜色：`#00FF41`
- 从 `00:00` 开始向上计时，精确到秒

### 3.3 状态机

```
开机
    ↓ 自动
开始录制 → 红点闪烁，A/B 双线滚动，计时器启动，写入 SD
    ↓ Enter
结束录制 → 写 WAV 头并回列表
    ↓ Space
暂停录制 → 停止写入 SD、计时冻结 → Space 继续
    ↓ Esc
结束录制 → 写 WAV 头并息屏
```

## 4. 播放界面

### 4.1 界面布局

```
┌──────────────────────────────────────┐
│                           65%        │  ← 顶栏（仅电量，无标题）
├──────────────────────────────────────┤
│                                      │
│   [波形进度条 / 静态波形图]            │  ← 波形+进度区 80px 高
│   ──────────┼──────────────────────  │  ← 中轴线
│                                      │
├──────────────────────────────────────┤
│ 03:34                                │  ← 底栏左下时间
└──────────────────────────────────────┘
```

### 4.2 波形进度条

这是播放界面最核心的功能组件，将传统进度条升级为可视化波形进度条。

#### 实现原理
- 打开 WAV 文件后，后台按 `WAVE_BARS` 个小柱快速抽样
- 每个小柱取该区间 3 个 256-sample 小片，并保留更大的 **绝对波峰**，归一化到 0–1
- 绘制为**以中轴线为基准、上下对称的竖线**（类似 ECG 风格）

#### 视觉规格
- 已播放 / 未播放部分：统一亮绿 `#00FF41` 小波形柱，播放头负责表达当前进度
- 播放头（playhead）：1px 宽亮白竖线 `#FFFFFF`，随进度移动
- 中轴线：1px `#003300`，贯穿全宽

#### 刷新策略
- 播放入口不做完整 WAV 预扫描，优先启动声音。
- A线在扬声器队列空隙里按 60 柱快速抽样生成，播放缓冲不再二次改写 A线，避免播放头前后形态变化。
- A线画成和录制 A线同款的 60 个竖向小波形柱；空段只保留暗基线，播放头负责区分当前位置。
- 播放头随播放缓冲推进刷新，控制在音频队列可承受范围内。
- 按键必须先反馈再执行：列表播放先显示 `PLAY`，播放中切换先显示 `PREV/NEXT`，暂停/继续先显示 `PAUSE/PLAY`，不让用户猜测按键是否生效。
- 播放中上下切换复用已有录音列表缓存，避免每次切换前重新扫描 SD。
- 录音保存回列表复用录音开始时顺手填充的列表缓存；列表删除后只更新内存列表，不重新扫描 SD。

### 4.3 时间显示

- 格式：`MM:SS`（当前播放位置）
- 位置、字号与录音界面相同

### 4.4 磁带轮动画（可选，原稿提及）

- 在波形区右侧或底部居中绘制两个小圆圈，模拟磁带卷轴
- 播放中顺时针慢速旋转（每秒约 1/4 圈）
- 快进时加速旋转
- 停止时静止
- 实现方案：绘制带缺口的圆弧（2个弧段），每帧旋转偏移角度

### 4.5 导航
- 右侧标签栏已取消，播放页使用 240px 全宽内容区。
- 播放自然结束后自动返回列表。
- 回车暂停/继续播放；退格返回列表；;/.切换上一条/下一条录音。反斜杠不再作为主流程按键。

### 4.6 按键行为

| 按键 | 行为 |
|------|------|
| 回车 | 暂停/继续播放 |
| 退格 | 返回列表 |
| ;/. | 切换上一条/下一条录音 |
| 长按右方向键 | 快进（2×/4× 速，松开恢复）|
| 长按左方向键 | 倒带（反向 2× 速，或按段回跳）|
| 短按右方向键 | 进度 +5 秒（可选）|
| 短按左方向键 | 进度 -5 秒（可选）|
| 空格 | 进入录音并立即开始写入 |
| Esc | 停止播放并息屏 |
| 长按 DEL 1.2秒 | 底部红色进度条走满后删除当前录音，长按时显示“删除中”，松开取消 |

#### 快进/倒带视觉反馈
- 快进时：磁带轮加速旋转，播放头加速移动
- 波形区顶部短暂显示 `▶▶` 或 `◀◀` 图标（500ms 后消失）

## 5. 列表界面

> 原稿标注「你来完善」，以下为补充设计建议，可接受修改。

### 5.1 界面布局

```
┌──────────────────────────────────────┐
│ 录音列表                  65%        │  ← 顶栏，标题 + 电量
├──────────────────────────────────────┤
│ > [001] 2024-01-15  03:34  ████░░░  │  ← 选中行，高亮
│   [002] 2024-01-15  01:22  ██░░░░░  │
│   [003] 2024-01-16  00:45  █░░░░░░  │
│   [004] 2024-01-16  05:12  ███████  │
│   [005] 2024-01-17  02:08  █████░░  │  ← 最多显示 5 行
├──────────────────────────────────────┤
│ ↑↓选择  回车播放  长按DEL删除        │  ← 底部操作提示栏
└──────────────────────────────────────┘
```

### 5.2 列表行设计

每行高度：**22px**，可显示约 5 行。

| 元素 | 内容 | 宽度 | 颜色 |
|------|------|------|------|
| 序号 | `[001]` | 32px | 暗绿 |
| 文件名/日期 | `rec_001` 或从文件名解析 | 80px | 亮绿 |
| 时长 | `03:34` | 36px | 亮绿 |
| 迷你波形预览条 | 7 格小方块 ████░░░ | 48px | 亮绿/暗绿 |

- **选中行**：整行填充暗绿背景 `#003300`，文字高亮 `#FFFFFF` 或 `#00FF41`
- **选中游标**：`>` 符号在行首

### 5.3 滚动行为
- 列表超过 5 项时滚动显示
- 选中项始终保持在第 2–4 行（居中对齐，避免贴顶/贴底）
- 右侧可加 1px 宽滚动条（亮绿色细线，按比例）

### 5.4 删除流程

```
列表界面（正常模式）
    ↓ 长按 DEL
底部红色进度条从 0% 走到 100%
    ↓ 满 1.2 秒
显示 `DEL`，文件删除，内存列表刷新，光标移到上一项
    ↓ 中途松开
取消删除，列表保持不变
```

- 删除手势统一：录音中、播放页、列表页都为长按 `DEL` 1.2 秒。
- 长按过程中显示红色进度条和“删除中”；未满 1.2 秒松开即取消。
- 列表删除不再完成后重新扫描 SD；只在内存列表移除该条，必要时才重写快捷键绑定。
- 重要录音独立存放在 `/IMPORTANT`；绑定快捷键或在列表按 `Ctrl+Enter` 会将文件从 `/REC` 移入 `/IMPORTANT`，列表中重要项显示 `*`。清理普通 `/REC` 不影响重要录音。

### 5.6 导航
- 右侧标签栏已取消，列表页使用 240px 全宽内容区。
- 回车播放选中录音；Esc 退出列表并息屏，之后按空格唤醒会直接录音，按回车唤醒会回到列表。

## 7. 动画与特效说明

### 7.1 红点闪烁（录音中）
- 频率：**1Hz**（亮 500ms → 灭 500ms）
- 实现：`millis()` 取余，奇偶切换绘制/不绘制

### 7.2 录音波形滚动
- 每帧向左平移 2px，新数据从右侧补入
- 使用帧缓冲或逐列移位绘制（不 `fillRect` 清屏，用 `scroll` 或逐像素移位）

### 7.3 磁带轮旋转
- 绘制：两个圆（左轮/右轮），每个圆内画 3 条辐射线（120° 间隔）
- 状态：停止/播放/快进，分别对应不同角速度
- 实现：存储当前角度变量 `float reelAngle`，每帧累加 `Δθ`，绘制旋转后的辐射线

### 7.4 删除进度条
- 出现位置：底部右侧，不遮挡左侧计时器。
- 交互：按住 `DEL` 后红色进度条从 0% 走到 100%，并显示“删除中”；满 1.2 秒执行删除。
- 取消：未满 1.2 秒松开即取消，进度条消失。

### 7.5 快进/倒带图标
- 位置：波形区顶部中央
- 内容：`>>` 或 `<<`，字号 font2
- 出现时机：开始长按时显示，松键后 500ms 消失

### 7.6 界面切换
- 不做复杂动画（资源有限）
- 切换时执行一次 `fillScreen(BLACK)` 全清屏即可
- 可选：做 2 帧的扫描线效果（从上到下逐行清除）

## 8. 优化建议（超出原稿）

以下为在原稿基础上的设计优化，实现时可选采纳：

### 8.1 文件名显示优化
- 列表行中如果文件名过长，做**横向滚动**（选中行自动 marquee）
- 或改用「recNNN」统一序号显示，右侧标注文件时长

### 8.2 播放界面增加文件标题行
- 波形区顶部叠加显示当前文件名（小字，暗绿色），不影响波形

### 8.3 电量动态图标
- 在百分比左侧加 3×6 px 的电池图标（3 格像素块），按电量填充格数
- 低电量时整个电量显示区红色闪烁

### 8.4 录音界面增加文件序号预告
- 顶栏 "REC" 右侧显示将要保存的文件号，如 `REC #042`
- 帮助用户了解录音存在哪里

### 8.5 列表界面迷你波形预览
- 每行右侧 7 格小方块代表该录音的振幅分布（粗略缩略图）
- 打开列表时后台异步读取（或复用已缓存的波形数据）

### 8.6 静音标记（可选高级功能）
- 不恢复“自动暂停”逻辑，避免增加录音状态分叉。
- 如需提示静音，可只做轻量视觉标记，不改变录音写入流程。
