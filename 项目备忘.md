## 🔖 续接说明(开新窗口先看这段)

**整机已基本完成且稳定**:开机→空格录音(绿色声波+W/S调音量)→停止→回车播放→N降噪。录音文件在电脑上听是干净的。

**⚠️ 当前有一处刚改、等用户验证(最重要)**:
为消除"开机爆音",最新一版把 `cfg.internal_spk = false`(开机不启用扬声器),并**手动配置扬声器**(setup 里设引脚 41/43/42、I2S_NUM_1)+ **playWav 里手动写 ES8311 DAC 寄存器**(0x00=80,01=B5,02=18,0D=01,12=00,13=10,32=BF,37=08)开启 DAC。
- **要确认两点**:① 冷启动开机不再爆音;② **按回车播放是否还有声音**(手动配 DAC 若有误会变没声音)。
- **若播放没声音了**:把 `internal_spk` 改回默认(删掉 `cfg.internal_spk=false` 和手动 speaker.config、删掉 playWav 里手动 DAC 那段、并在 setup 预热末尾恢复 `Speaker.begin()`),即回到"有声音但开机会爆音"的稳定版。仓库 `toolbox/toolbox.ino` 是当前(internal_spk=false)版本。

**音频状态机(当前设计, 为了少爆音)**:
- 开机: 不开扬声器(无开机爆音), 麦克风预热一次(空跑录音会话, 修冷启动首录音量小/断续)。
- 录音: 只用麦克风。**停止录音不关麦**(避免停止爆音), 麦保持开着。
- 播放(playWav): 这时才 `Mic.end()` + `Speaker.begin()` + 手动开 DAC → 播放 → 不关扬声器。
- 仍存在的小问题: **播放前一下"功放上电爆音"**(最难消, 紧挨声音、较自然, 暂留)。

**爆音问题进展**: 停止录音爆音=已修(停止不关麦); 设备播放"哒哒"=已修(播放改双缓冲, 之前单缓冲被覆盖); 开机爆音=最新版尝试用 internal_spk=false 消除(待验证)。

**2026-05-24 录音音质/按键摩擦处理结论**:
- 当前可用版本保留 GitHub 基线实时录音链路: `REC_RATE=16000`, `REC_N=256`, `recGain=50`, `mc.magnification=1`, `mc.noise_filter_level=3`, `processMicBuffer()` 仍是软件增益 + `lpf` 系数 `160` + `tanhf` 软限幅。
- 新增保存后轻量按键摩擦处理 `suppressKeyFriction(path, effData)`: 录完写好 WAV 后扫描一遍 PCM, 只处理"低 RMS + 高相邻采样变化量"的帧。
- 样本来源: `REC_0399.wav`=纯按键/手指摩擦, `REC_0400.wav`=说话夹杂摩擦。分析结论: 摩擦声 RMS 低(A线不明显), 但高频比例/相邻采样变化量高; 人声帧 RMS 高、频谱重心更低。
- 当前规则: `rms < 1900 && avgDiff > 140 && avgDiff*256 > rms*90` 或 `rms < 2200 && avgDiff > 300` 判为摩擦; 命中后 `hold=2` 个缓冲。
- 当前处理方式: 不压整块音量, 只削高频刮擦分量。每帧做一阶低通 `lp += (sample-lp)*96/256`, 分离 `hi=sample-lp`, 输出 `lp + hi*64/256`。这样比旧方案自然, 不会把说话变成断断续续。
- 明确踩坑: 不要再用强实时 AGC/强语音门/长时间整块衰减处理摩擦或碰撞; 这些会让人声变糊、断续, 且按键摩擦不一定会被 A线/RMS 捕捉到。

## 一、这是什么
给 M5Stack **Cardputer-Adv** 写的一个"工具箱"固件(Arduino/C++)。
- 风格: 纯黑底 + 黑客帝国荧光绿 + 中文界面。
- 主页: `[空格] 录音`  `[回车] 播放最近`。
- 录音机: 实时绿色声波(示波器), 录音中 **W 调大 / S 调小** 音量, **其他键停**。
- 录完结果页: `[回车] 播放`  `[N] 降噪`(FFT 谱减法)  其他键返回。
- 录音存成 16kHz 单声道 WAV, 放在 SD 卡 `/REC/REC_xxxx.wav`。

## 三、怎么编译 + 烧录(核心流程)
1. 用**能传数据的 USB-C 线**把机器连到电脑(劣质/充电线会"无法识别 USB", 换线!)。
2. 电源开关 ON。
3. 在 PowerShell 里跑(端口一般是 COM3, 不确定先用 `arduino-cli board list` 查):
```
$cli="C:\cardputer\tools\arduino-cli\arduino-cli.exe"
& $cli compile --fqbn esp32:esp32:m5stack_cardputer C:\cardputer\sketches\toolbox
& $cli upload  --fqbn esp32:esp32:m5stack_cardputer -p COM3 C:\cardputer\sketches\toolbox
```
- **开发板型号(FQBN)**: `esp32:esp32:m5stack_cardputer`(库会自动识别 Adv)。
- 如果烧不进/认不到: 进**下载模式** = 电源 OFF → 按住 G0 → 通电 → 松开 G0, 再烧。
- 看程序崩溃日志: 用串口工具连 COM3 @115200, 会打印 `Guru Meditation` 等。

## 四、关键的坑和决定(很重要, 别重复踩)
1. **核心必须用 esp32 3.1.3, 不能用 3.3.x!**
   - 3.3.8(最新)上**麦克风录不了音**(录到静音/平线), 这是新核心的 I2S 录音回归 bug。
   - 3.1.3(IDF 5.3)麦克风正常。降级方法见 `C:\cardputer\prefetch313.ps1`。
2. **键盘库被改成"轮询"了**(去掉了 GPIO 中断)。
   - 文件: `...\M5Cardputer\...\KeyboardReader\TCA8418.cpp`。
   - 原因: 原版用 GPIO 中断, 在 3.1.3 上开机会撞爆系统 ipc1 任务栈 → 不停重启/闪屏。
   - ⚠️ **如果重装 M5Cardputer 库, 这个改动会丢, 要重新改**(把 begin() 里的 attachInterrupt 删掉、update() 改成 `while(available()) getEvent...` 轮询)。
3. **开机会"预热"麦克风**(空跑一次录音会话 + 暖一下 SD), 否则**冷启动后第一次录音音量小、断续**。
4. **国内下载慢**: esp32 核心用 Jihulab 镜像 + 从 `dl.espressif.cn` 用 curl 高速预取到缓存, 再让 arduino-cli 从缓存装(arduino-cli 自带下载器对国内 CDN 很慢)。
5. **这颗芯片(ESP32-S3FN8)没有 PSRAM** → 跑不了 Espressif ESP-SR 专业降噪。所以只能用自写的轻量谱减法降噪。
6. 喇叭是 1W 小喇叭, 本身就小声; **要大声清晰用 3.5mm 耳机**。

## 五、可调参数(都在 toolbox.ino 里)
| 参数 | 位置 | 作用 |
|---|---|---|
| `recGain`(默认 50) | 文件顶部 | 录音音量(也可录音时 W/S 现场调) |
| 低通系数 `160` | recorderApp 的 lpf 那行 | 越小越闷、压高频底噪越多 |
| `HEAD_SKIP`(13) | recorderApp | 掐头时长(去开始键声) |
| `tailTrim`(REC_RATE/4) | 收尾处 | 掐尾时长(去结束键声) |
| 软限幅 `tanhf` | 每采样处理 | 防大声"哒哒"爆裂 |
| `suppressKeyFriction()` | 保存后处理 | 按键/手指摩擦抑制: 低 RMS + 高 avgDiff 帧只削高频, 不整块降音量 |
| 降噪 `ALPHA`(3.0)/`BETA`(0.05) | noiseReduce() | 谱减强度/谱底; ALPHA 越大降噪越狠但易有"水声" |
| 喇叭 DAC `0x32` | playWav() | 回放音量微调 |
